GTD

目的：把大脑的想法分离出来,使大脑可已更加集中精神的处理一件事情
好处: 知道下一步要做的事情,
假如有成千上万个想法在那里嗡嗡乱转，把它们写在纸上，我感觉会好得多。一旦变成文字记录下来，我就不再担心会忘记

建议使用便利贴, 不要完全依赖软件,原因之一是使用便利贴可也保持使用状态的一致性, 即手上拿着笔可马上写, 而不用经过解锁手机打开软件的步骤添加. 其二就是可以有效的避免了手机其他因素的干扰, 随也不可能知道手机什么时候回有消息过来.

使用便利贴可以记录: 名词解释,

基本原则

  • 知道下一步要做什么

    感觉有很多事情要做, 但是又不知道从何开始, 所以说确定要做的事情必须有优先级, 且事情分得足够小, 以至于可以在一个番茄钟内完成

  • 控制做的事情的时间范围

    不放回原则: 当决定开始某项任务时就尽力完成, 不要在放回收集箱里
    把类似的任务放在一起。

  • 提高注意力, 不要分心, 一次只做一件事, 不要多项工作同时进行

    尤其是牵涉到学习的最好就单任务, 特别是在啃干货的时候, 不能自己增加环境影响. 尽量不要为了玩或者拖延,将工作放在最后一起做. 因为这时往往要加班加点, 就好像为了买超市超出当前购买力的东西,你不得不为此支付利息,需然利息不多,但是会减缓未来的成长

同时做两件事儿的诀窍在于

  1. 其中一件事儿最好是机械的,少点技术含量的;「运动」
  2. 另外一件事儿不是特别需要大量脑力支持的;「社交，看新闻」
  3. 两件事儿都应该是确实有效用因而值得时间投资的。

  • 每周回顾清单

    当你的系统中存放着越来越多的任务时,你就会丧失整理得欲望

    回顾的时候做什么呢?

  • 加工收集的所有的“材料” (脱水)，尽量整合进自己的知识系统，保持「自洽」性

  • 回顾和检查你的系统
  • 更新各类清单
  • 做到清洁，清楚，实时，完整
  1. 按照一定的关联性收集文件(首字母,某类事情)
  2. 不要将资料塞满柜桶,2/3就差不多了,以免资料太多难找
  3. 清单要放在自己随处可见的地方,方便自己随时检查与使用
  4. 基本的应用场景都应该有对应的清单

GTD工具的选择

对于GTD理论来说,必须具有的功能就是

  • 构件清单列表
  • “工作任务”清单
  • “下一步行动”清单
    • “待处理”清单
  • 为事件添加优先级
  • 有提醒功能
    • 按执行的时间分
      1. 执行
      2. 待办/将来会做的事
    • 按使用场景分
      1. 在办公室
      2. 在路上
      3. 在电脑旁

仪式感

经常感到焦虑，只因为自己做的事情看不到任何正向的结果，但是又想坚持下去. 有很多想做的事情，但是没有优先级，任务都是突然兴起加的. 所以说要有仪式感来强迫自己做某些事情

GTD使用流程

怎么寻找任务

每天晚上将第二天必须完成的 3~5 个任务列出来,然后按优先级完成.

任务的收集

  1. 检查自己的工作环境,是否有工具要替换或升级
  2. 感觉效率低的地方记下来
  3. 待办的事情
  4. 某一时刻的想法

结果导向型.

很多新的任务是基于上一个任务的结果才会出现的, 所以有些东西接下来会怎样还不得而知.那么该怎么处理这些东西呢?

既然是未知的东西, 那么先让它运行起来, 到时候看情况在做决定.
